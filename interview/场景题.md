# 场景题

## 如果线上出现了GC频繁，会出现什么反应？你的排查全链路是什么？怎么解决？

**线上表现（症状）**

| 现象            | 说明                                  |
| --------------- | ------------------------------------- |
| 接口响应变慢    | STW（Stop The World）导致请求延迟飙升 |
| CPU飙高         | GC线程占用大量CPU资源                 |
| 内存持续增长    | 堆内存无法有效回收，逼近OOM           |
| 频繁Full GC     | 老年代回收压力大，每次耗时长          |
| 服务不可用/重启 | OOM Killed，容器/进程崩溃             |
| 告警轰炸        | JVM监控、APM告警持续触发              |

**全链路排查流程**

```tex
线上GC频繁
     │
     ├─ 第1步：快速定位 → 看监控指标
     │
     ├─ 第2步：GC日志分析
     │
     ├─ 第3步：堆内存分析 → Heap Dump
     │
     ├─ 第4步：代码级分析 → 找根因
     │
     └─ 第5步：解决 + 验证
```
